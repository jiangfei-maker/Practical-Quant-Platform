# 实战量化交易平台 - 全面优化计划 (Optimization Plan V2)

基于对当前项目结构、代码质量及功能实现的全面评估，制定以下分阶段优化计划。旨在提升平台的稳定性、数据准确性、策略丰富度及用户体验。

## 一、 核心数据与基础设施增强 (Data & Infrastructure)

**目标**：确保数据源稳定、准确，为上层应用提供坚实基础。

1.  **数据源健壮性提升**
    *   [ ] **多源容灾机制**：在 `FinancialDataFetcher` 和 `MarketFetcher` 中引入备用数据源（如从 AkShare 切换到 Tushare/Baostock 的降级方案），防止单一接口失效导致服务不可用。
    *   [ ] **数据质量卫士 (Data Guard)**：完善 `core/data/data_guard.py`，增加对异常值（如价格为负、成交量为0、PE/PB 极端值）的自动清洗和报警。
    *   [ ] **缓存策略优化**：在 Streamlit 和数据层之间实施更智能的缓存（Redis 或本地文件缓存），减少对外部 API 的频繁请求，提升页面加载速度。

2.  **数据库性能优化**
    *   [ ] **DuckDB 结构优化**：检查 `quant_core.duckdb` 的表结构，为高频查询字段（如 `ts_code`, `trade_date`）添加索引。
    *   [ ] **并发读写处理**：确保 `DBManager` 在多线程/多进程环境下的安全性，防止数据库死锁。

## 二、 功能模块深度优化 (Module Deep Dive)

### 1. 策略回测 (Strategy Backtest)
**现状**：仅支持双均线和 RSI 策略，指标展示基础。
**优化计划**：
*   [ ] **回测报告升级**：引入专业的绩效指标展示（夏普比率、最大回撤、索提诺比率、胜率、盈亏比），并绘制资金曲线、回撤曲线和月度收益热力图。
*   [ ] **策略参数动态化**：支持用户在 UI 上动态添加策略参数，而非硬编码在页面中。
*   [ ] **批量回测**：支持对整个板块或自定义股票池进行策略扫描，筛选出表现最好的标的。
*   [ ] **交易细节模拟**：加入滑点 (Slippage) 和手续费 (Commission) 设置，使回测结果更贴近实战。

### 2. 大盘分析 (Market Analysis)
**现状**：主要展示核心指数，维度较单一。
**优化计划**：
*   [ ] **市场情绪看板**：增加涨跌家数比、涨停/跌停家数统计、市场连板高度监控。
*   [ ] **资金流向监控**：集成北向资金、主力资金流向数据，展示行业板块资金流入流出排行。
*   [ ] **行业热力图**：使用 Treemap 展示各行业板块的涨跌幅和成交量占比。

### 3. 财务分析 (Financial Analysis)
**现状**：已修复基础数据，但分析维度可扩展。
**优化计划**：
*   [ ] **杜邦分析可视化**：将杜邦分析的 ROE 拆解通过图表直观展示。
*   [ ] **同行对比**：在查看某只股票时，自动拉取同行业龙头的财务数据进行对比（PE/PB Band, 利润增速对比）。
*   [ ] **估值模型**：引入 DCF（现金流折现）或 PEG 估值模型，提供理论股价参考。

### 4. AI 投研 (AI Research)
**现状**：具备基础框架。
**优化计划**：
*   [ ] **研报智能摘要**：利用 LLM 对长篇研报进行关键信息提取（观点、评级、目标价）。
*   [ ] **智能问答增强**：优化 RAG 检索逻辑，支持针对特定股票的自然语言问答（如“茅台去年的净利润增长原因是什么？”）。

## 三、 UI/UX 与工程化体验 (Engineering & UX)

1.  **用户体验统一**
    *   [ ] **全局配置中心**：在 `系统管理` 页面集中管理 API Key、回测默认参数、UI 主题等。
    *   [ ] **交互反馈**：在数据加载、回测计算等耗时操作中增加进度条和状态提示。
    *   [ ] **响应式布局**：优化图表在不同屏幕分辨率下的展示效果。

2.  **代码质量与测试**
    *   [ ] **单元测试覆盖**：为核心计算逻辑（如指标计算、回测引擎）编写 pytest 测试用例，确保逻辑正确。
    *   [ ] **类型提示 (Type Hinting)**：全面完善 Python 类型注解，减少运行时错误。
    *   [ ] **日志规范化**：统一日志格式，便于排查线上问题。

## 四、 近期执行路线图 (Roadmap)

1.  **本周重点**：完成 **策略回测模块** 的升级，支持详细的回测报告和交易成本模拟。
2.  **下周重点**：完善 **大盘分析** 模块，增加市场情绪和资金流向监控。
3.  **后续**：逐步推进 AI 投研和深度财务分析功能。
